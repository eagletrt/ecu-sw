============================== CONTEXT ==============================
The first FSM version had some issues:
- Only one global error state (`fatal_error`), reachable only until `idle`.
- No local error states, which made debugging ambiguous.
- `idle` was not clearly defined.
- Manual vs. Autonomous System (AS) driver were not differentiated.
- Transition labels were missing or unclear.

============================= NEW VERSION ===========================
The new FSM version addresses these issues:

- Added local error states:
   - `precharge_error`
   - `as_init_error`
   This makes it clear where an error originated.

- Kept `fatal_error` and `shutdown` for global safety.

- Clearer driver separation:
   - Manual driver -> enables inverter directly.
   - Autonomous driver (AS) -> requires extra init checks.
   If AS init fails, it goes to `as_init_error`, which can still
   fall back to manual drive.

- Precharge/discharge phases remain mandatory for safety.
   - Any failure during precharge -> `precharge_error -> discharge`.
   - Any shutdown goes through discharge when possible.

- `re_enable_inv_drive` is kept, as the code requires it.

============================== REMAINING POINTS =====================
- `idle` still serves as the "system on, ready but not driving" state.
- Transition labels in the DOT file must remain simple
  (or removed) for code generation, but they are useful for visualization.
- A `poweroff` state could be added in future if shutdown handling
  needs more granularity.
